適用の優先順位（要約）

§B（個別ルール群） は §A（全依頼ルール群） より優先する。食い違うときは、該当する依頼・工程・機械の組み合わせでは §B を採用。

§B 群内の強さ: B-1 を最優先 → B-2 と B-3 は同一 b_tier = 1 帯（EC 先行と未着手の検査／巻返し）。グローバル task_queue.sort では、加工途中・納期基準を見たうえで B-1 → B-2/B-3 帯 → その他 の順。

本文セクション対応: (1) §B 個別の追加ルール (2) §A 全加工依頼への追加ルール (3) 「1. 配台の対象外」以降の一般ルール。

§A全加工依頼に適用する追加ルール

A-1 「加工内容」列に基づく工程順序（同一依頼NO）

データ: 「加工計画DATA」／「配台計画_タスク入力」の列 加工内容（段階1で継承）。

  • 同一 依頼NO で複数行があり、いずれも配台不要でないとき、加工内容を カンマ区切りした順が工程の正しい順序（先頭が最初の工程）。
  • 先行工程に 残ロールがある間、後工程には _task_blocked_by_same_request_dependency により割付を試みない（§A-1）。
  • 工程名は分割要素と NFKC・空白除去で照合。加工内容に現れない工程名の行にはこの順序制約はかからない（結果シートでは不整合時にタスクID赤表示など）。
  • §B-2 / §B-3 の後続は、枠・占有など別チェーン（B-2.6 等）が先に効く場合がある。
A-2 同一依頼NOの直列化（rank が無い行・加工内容に載らない工程）

目的: 同一依頼NOで異なる工程（別行）は、別設備でも 同時に加工しない（流れを1本化）。

  • §A-1 で両行に process_sequence_rank が付くときは rank のみで前後判定（本節は追加しない）。
  • どちらか一方でも rank が無い ときは、シート上で上にある行（same_request_line_seq が小さい方）の残ロールがゼロになるまで、下の行には割付を試みない。

実装: build_task_queue_from_planning_df、_task_blocked_by_same_request_dependency

§B個別に指定する追加ルール

§B は本文の一般ルール（§1 以降）より優先。§A と食い違う場合も該当組み合わせでは §B。

B-1 — 検査（熱融着機　湖南）× 加工途中 → 配台試行順を優先

対象: 工程 検査 × 熱融着機　湖南 かつ in_progress（実績あり）。

  • グローバルソートで b_tier = 0。B-2/B-3 帯より先に並び得る。
  • 日内キーでは dispatch_trial_order を process_sequence_rank（§A-1）より前に置く。加工途中の検査では試行順が EC 先行の §B-2 段より先に効く。
  • B-1 は B-2 より常に先にソートへ反映。
B-2 — EC（EC機　湖南）→ 検査（熱融着機　湖南）ロールパイプライン

先行 EC を全日で進め、検査はロール枠・二相パイプライン・リワインド等で制御。§B-1 の次に EC と未着手検査を b_tier = 1 帯にまとめ、帯内は EC を検査より先。

段階工程名機械名
先行ECEC機　湖南
後続検査熱融着機　湖南

関連（例）: _roll_pipeline_inspection_assign_room、ROLL_PIPELINE_INITIAL_BUFFER_ROLLS、PLANNING_B1_INSPECTION_EXCLUSIVE_MACHINE、PLANNING_B2_EC_FOLLOWER_DISJOINT_TEAMS、B-2.8 工程間WIP WIP_LIMIT_EC_BEFORE_INSP_ROLLS ／ 集計 WIP_LIMIT_EC_BEFORE_INSP_AGGREGATE。

B-3 — EC（EC機　湖南）→ 巻返し（EC機　湖南）ロールパイプライン

B-2 と同系のロールパイプライン。後続を 巻返し × EC機　湖南 とした版。§B-2.2〜B-2.7 の「検査」を「巻返し」に読み替え可能。

段階工程名機械名
先行ECEC機　湖南
後続巻返しEC機　湖南
B-4 — スリット（スリット機1　湖南）→ SEC（SEC機　湖南）

目的: SEC 前のスリットばかり蓄積しないよう、SEC を相対優先。

  • WIP（総ロール） max(0, スリット完了合計 − SEC完了合計) が上限以上なら スリットを新規配台しない。環境変数 WIP_LIMIT_SLIT_BEFORE_SEC_ROLLS（既定 20、0 以下で無効）。
  • B-4.1: 加工内容が スリット→SEC の依頼で、同一依頼内（スリット完了−SEC完了）が閾値未満の間は SEC を候補から除外。閾値 SLIT_BEFORE_SEC_MIN_SLIT_ROLLS（既定 5）。
  • B-4.2: 自動算出の配台試行順では、同一依頼内で スリット行の直後に SEC 行が来るよう隣接化。
B-5 — 特別ルール列挙（速度・人数・日内優先）※抜粋

詳細は リポジトリ直下の 特別ルール列挙.md。実装済みの例:

  • L2 スライス×スライス機1　湖南（100m 時 3名 or 速度フォールバック）
  • L3 EC×EC機　湖南（製品名 NR28 で必須3名）
  • L4〜L7 SEC×SEC機　湖南（幅・依頼NO 条件で速度や最低人数）
  • L8 接続×熱融着機　湖南（製品長 105m で速度 20m/分）
  • L9 スライス近傍優先（※実装状況は正本参照）
  • PN優先（SEC同日） など
B-6 — 接続（熱融着機　湖南）→ SEC（SEC機　湖南）

目的: B-4 と同趣旨で 接続完了前に SEC が先行しない。

  • WIP が上限以上なら 接続を新規配台しない。WIP_LIMIT_CONNECTION_BEFORE_SEC_ROLLS（既定 20）。
  • B-6.1: 加工内容 接続→SEC で、同一依頼内の差が閾値未満の間は SEC を除外。CONNECTION_BEFORE_SEC_MIN_CONNECTION_ROLLS（既定 5）。
  • B-6.2: 自動試行順で 接続行の直後に SEC 行が来るよう隣接化。
