modGanttDateNav（設備ガント：シート上コンボで日付ジャンプ）

【前提】
・共通定義.bas に SHEET_RESULT_EQUIP_GANTT / SHEET_RESULT_EQUIP_GANTT_ACTUAL_DETAIL があること。
・VBA ツール → 参照設定 で「Microsoft Forms 2.0 Object Library」にチェックが入っていること（未チェックだとコンパイルエラーになります）。

【インポート】
1. clsGanttDateNavCombo.cls をインポート（クラス モジュール）。
2. modGanttDateNav.bas をインポート（または差し替え）。
3. ユーザーフォーム UF_GanttDateNav は不要です。プロジェクトに残っていれば削除して構いません。
4. デバッグ → VBAProject のコンパイル。

【使い方】
・コンボは **A1 セルの左上** に幅 A:B 列分で配置します（Python 出力の行1タイトルは **C1 から** 始まるため重なりません）。
・「結果_設備ガント」または「結果_設備ガント_実績明細」を表示した状態で、マクロ「結果_設備ガント系_日付ジャンプコンボを確保」を実行する。
・**段階2** の取込完了処理（業務ロジック）の中で「結果_設備ガント系_日付ジャンプコンボを両シートで確保」が自動実行され、openpyxl でシートが差し替わって消えたコンボが再作成されます。
・コンボで日付を選ぶと、その日ブロック先頭行の列 A がアクティブになります。
・手動で日付ブロックだけ変えたときは「結果_設備ガント系_日付コンボを再充填」でリスト更新可。

【補足】
・コンボの OLE 名は GanttDateNavCombo 固定です。手動で削除したあと再実行で再作成されます。
・ASCII の短いパスにプロジェクトを置くと OLE 周りのエラーが減ることがあります。
