オンラインマニュアル（慣れた人向け）

よく使う導線とつまずきどころだけを短く記載しています。

はじめに・全体像

  • メインの往復は 実行・ログ ? 結果系ビュー。
  • タブの並びは「タブ整理」で好みに合わせ可能（セッションに保存）。

実行・ログ

  • ログはフィルタで ERROR のみに絞るとトラブル時が速い。
  • 段階実行中は他タブが無効になるため、長時間ジョブの前に設定タブを済ませる。
  • 実行・ログを開く

環境設定（環境変数・メモリ・グローバル）

ネットワークドライブのソースはキャッシュ（.pm-ai-cache）とセットで理解するとトラブル解析が速いです。

配台計画_タスク入力

ブックを開き直したあと段階1へ。シート選択状態はセッションに載る場合があります。

マスタ読込サマリ

段階1前後で差分が出たら、マスタ側のパス・タイムスタンプを疑う。

段階1（実行とログの見方）

ログ末尾のエラーコードより、先に入力パスと PM_AI_* を確認。

段階1 成形結果

ここで異常なら段階2に進まず入力／マスタへ戻る。

段階2（計画シミュレーション）

Excel 書き出しの要否・フォントは実行タブ側のオプションとセットで確認。

計画結果ビューア・結果_配台表

JSON の場所が分からないときは計画結果ビューアの「設定」アコーディオンからパスを確認。

配台計画手動修正

内側タブを切り替えて「タスク×日付」と「工程+機械×日付」を使い分け。

設備ガント・納期管理ビュー・オペレーターカード

納期管理ビューは子タブが多いので、タブ整理ツリーで名前と対応を確認すると良いです。

特別ルール・配台不要ルール・実績DATA

特別ルールは Markdown 要約と列挙ファイルの整合に注意（リポジトリ運用）。
