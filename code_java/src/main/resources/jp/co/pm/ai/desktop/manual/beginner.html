はじめに・全体像

このアプリは、配台計画に関する タスク入力の取り込み、段階1（前処理・成形）、段階2（計画シミュレーション）を Python スクリプトで実行し、結果を表や JSON、ガント等で確認するためのデスクトップ UI です。

最上段のタブを左から順に辿ると、典型的な作業の流れに近い配置になっています。初回は 実行・ログ でパスを確認し、配台計画_タスク入力 でブックを開いてから、再び 実行・ログ へ戻り段階1→段階2の順に進めると迷いにくいです。

（補足）右端の「タブ整理」では、タブのグループ分けや色の調整ができます。業務の説明より UI の好みに関わるため、まずは既定のままで問題ありません。

実行・ログ

ワークブック欄にはタスク入力 Excel のパスを指定します。空欄のときは、環境変数と既定のブートストラップロジックでパスが決まります（ヒントは入力欄のプロンプト文言を参照）。

Python と スクリプトディレクトリ（通常はリポジトリの code/python）が段階1・段階2の実行に使われます。

段階1・段階2ボタンでそれぞれの Python が起動し、ログが一覧に流れます。ログはフィルタやフォントサイズで読みやすく調整できます。

注意：段階1または段階2が実行中は、ログ以外のメインタブは操作できません（処理の整合性のため）。進捗は画面上部のツールバーにも表示されます。

このタブを開く

環境設定（環境変数・メモリ・グローバル）

環境変数 タブでは、Python 子プロセスへ渡す変数やパス類を編集します。PM_AI_* が運用上の中心です。

メモリ設定 は JVM のヒープや監視表示に関わります。巨大ブックを扱う場合に調整します。

グローバル設定 は UI 全体の既定リセットなど、アプリ横断の設定です。

詳しい変数名は「詳細解説」深度や開発者向けの索引を参照してください。

配台計画_タスク入力

計画に投入するタスク入力ブックを開き、シート内容を確認・調整します。段階1の前提となるデータがここで固まります。

このタブを開く

マスタ読込サマリ

マスタ読込の結果を一覧・確認するためのタブです。入力データとマスタの整合に問題がないかを見る際に利用します。

このタブを開く

段階1（実行とログの見方）

実行・ログ の「段階1」ボタンで task_extract_stage1.py が実行されます。エラーが無ければ成形済みデータが後続の確認タブへ連携されます。

ログに失敗理由が出た場合は、環境変数・入力ブック・ネットワークソースのパスを優先して確認してください。

段階1 成形結果

段階1で出力された成形結果を確認します。問題なければ段階2へ進みます。

このタブを開く

段階2（計画シミュレーション）

実行・ログ の「段階2」で plan_simulation_stage2.py が実行されます。Excel への書き戻しオプション等は実行タブ上の設定を確認してください。

完了後、計画結果 JSON や配台表、各種ビューでアウトプットを追跡します。

計画結果ビューア・結果_配台表

計画結果ビューア (JSON) では multi_day やメンバー勤務など、計画結果 JSON をシート単位で確認できます。

結果_配台表 (JSON) では配台表形式の結果を扱います。

内部の子タブ名はタブ整理ツリー（「タブ整理」メインタブ）でも参照できるよう登録されています。

配台計画手動修正

自動計画の結果を起点に、タスク×日付や工程×機械×日付などの観点で手修正する画面です。内側に複数の子タブがあります。

このタブを開く

設備ガント・納期管理ビュー・オペレーターカード

設備ガント（グラフィック） で設備別のスケジュールを俯瞰します。

納期管理ビュー は計画と実績の比較などに利用します（内側に複数の子タブがあります）。

オペレーターカード は担当者単位の情報整理に使います。

特別ルール・配台不要ルール・実績DATA

特別ルール はリポジトリの説明テキストや列挙との対応を確認するためのタブです。

配台不要ルール (JSON) で除外ルール JSON を扱います。

実績DATA・明細の取得状態 で実績系ソースの状態を確認します。

業務ルールの文章の正はリポジトリの仕様書を参照し、実装の正はソースコードです。
