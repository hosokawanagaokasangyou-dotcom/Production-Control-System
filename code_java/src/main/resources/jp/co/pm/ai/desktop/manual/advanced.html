オンラインマニュアル（詳細解説）

画面項目とデータの対応、検証時の観点を記載します。業務仕様の文章はリポジトリ直下の仕様書を正とし、ここではアプリ操作に直結する範囲に絞ります。

はじめに・全体像

メインシェルは MainShellTabId のキーで永続化されます。タブ見出しのエイリアスは保存されますが内部キーは変わりません。

実行・ログ

段階1は task_extract_stage1.py、段階2は plan_simulation_stage2.py。子プロセスへの環境は環境変数タブの行集合から構築されます（空名・# 開始行は無視）。

実行中に他タブが無効になるのは MainShellController 側のガードです。

環境設定（環境変数・メモリ・グローバル）

PM_AI_TASK_INPUT_SOURCE_DIR / PM_AI_ACTUAL_DETAIL_SOURCE_DIR などフォルダ系はパス正規化の対象になる場合があります。ネットワーク不可時は起動時ログに警告が出ます。

配台計画_タスク入力

シート読込・列レイアウトは各コントローラの永続化設定（列順等）と相互作用します。不整合時は一度列フィルタ・並べ替え解除を試してください。

マスタ読込サマリ

マスタ Excel／CSV の参照結果がここに集約されます。パスは環境変数と実行タブの実効値を追うとよいです。

段階1（実行とログの見方）

ネットワークソースが見えない場合は .pm-ai-cache/network-source/ のメタとコピーを確認してください（チーム運用で Git 管理される場合があります）。

段階1 成形結果

成形結果は後段の JSON・Excel 出力の前提になります。シート単位で欠損が無いかを見ます。

段階2（計画シミュレーション）

段階2は計画コア（Python）側の設定・シート可視化オプションの影響を強く受けます。ログに出るパスで出力物を直接確認してください。

計画結果ビューア・結果_配台表

計画結果ビューアは JSON の複数シートをタブで開きます。内部タブ見出しは MainShellInnerTabCatalog とタブ整理ツリーの整合が取ってあります。

配台計画手動修正

機械 JSON の再読みなど長時間処理ではツールバー横の進捗表示が使われます。実行タブと同系統の UI です。

設備ガント・納期管理ビュー・オペレーターカード

納期管理ビューは複数ソース（計画・実績・アラジン加工計画等）を切り替えます。加工計画 DATA の解釈（換算数量・実加工数・未加工の組み合わせ）は実装ルールに従います。

特別ルール・配台不要ルール・実績DATA

配台に関する業務ルールの参照先としてリポジトリの 配台ルール.md、特別ルールの列挙・要約ファイルがあります。本文をマニュアルに複製せず、参照パスで誘導してください。
