2. 起動と基本操作

現場の一連手順（アラジン出力〜段階3〜アラジン転記）は 日常運用ガイド にまとめています。本章はアプリの部品と最短ルートの説明です。

アプリの起動

  1. 配布パッケージでは PMD.exe（実行ファイル名はビルドにより異なる場合あり）をダブルクリックします。
  2. 初回起動時、ポータブル配布では 環境変数の既定リセット が一度だけ行われることがあります（マーカーファイル削除後は再実行されません）。
  3. 起動後、ウィンドウが表示されれば正常です。失敗時は 実行・ログ タブまたはコンソール出力を確認します。

メインウィンドウの部品

部品 説明
テーマ 画面の明暗テーマ切替
進捗ラベル / 中断 段階1・段階2 実行中に表示。 中断 で Python 子プロセスを終了
Memory JVM ヒープ使用量の目安（メモリ設定タブと連動）
タブ行 各機能画面。実行中は段階1/2 以外が無効化されることがある

日常の基本手順

  1. 入力の確認 — 「配台計画_タスク入力」でブックパス・シート・加工計画 DATA の参照先を確認する。
  2. 段階1 — 「実行・ログ」から段階1 を実行し、成形結果・除外ルール JSON 等を得る。
  3. 段階2 — 同タブから段階2 を実行し、配台結果 JSON・xlsx・ガント用データを得る。
  4. 確認 — 「計画結果ビューア」「設備ガント」「結果_配台表」で内容を確認する。
  5. 手修正（任意） — 「配台計画手動修正」で編集し、保存後に再実行する。

実行・ログタブ

計画実行の起点です。段階1/2 のボタン、出力パスの表示、ログの追記がここに集約されます。

注意
段階1/2 実行中は他タブへの切り替えが制限されます。長時間処理の場合はログ末尾で進捗を確認し、必要なら 中断 で止めてから入力を見直してください。

タブの並べ替え

タブ整理 タブで、メインタブの順序・グループ・色を変更できます。変更はセッションに保存され、次回起動時も反映されます。

設定の保存

保存先 内容
~/.pm-ai-desktop/ ユーザーセッション（タブ配置、列幅、プロファイル等）
環境変数タブ PM_AI_* 系のキーと値（TSV/シート連携あり）
ユーザープロファイル UI 設定のエクスポート・インポート

終了

ウィンドウを閉じる前に、手動修正タブに 未保存 がないか確認してください。未保存のまま段階2 を実行しようとすると、ログに警告が出て実行が止まる場合があります。
