8. 困ったときは

段階1 / 段階2 が失敗する

  1. 実行・ログ の末尾 50 行付近を読む（エラーダイアログにも同内容が出る場合あり）。
  2. ログの 入力解決 行で、加工計画 DATA・配台計画ブックのパスが意図どおりか確認する。
  3. 環境変数 タブで TASK_INPUT_WORKBOOK / MASTER_WORKBOOK_FILE を確認する。
  4. ネットワークフォルダ利用時は 実績DATA・明細の取得状態 と .pm-ai-cache の更新を確認する。

「未保存のため実行できません」

  • 配台計画手動修正 … JSON を 保存 するか 再読み する。
  • 配台計画_タスク入力 … 表の編集を 保存 または 再読み してから段階2 へ。

タブが灰色で切り替えられない

段階1 または段階2 が 実行中 です。完了を待つか、ツールバーの 中断 で止めてから操作してください。

メモリ不足（OutOfMemoryError）

  1. メモリ設定 でヒープを増やし、アプリを再起動する。
  2. 計画期間・依頼件数を絞って再実行する。
  3. それでも失敗する場合はログと *.hprof の有無を開発担当へ共有する。

ガント・配台表が古い

段階2 完了後、実行タブの出力パスが更新されているか確認します。手動修正後は 保存 → 段階2 再実行 または結果 JSON の 再読み が必要です。

段階3 で当日数量が満たせない

段階3 は 勤怠・休憩・機械カレンダー に拘束されます。docx 原稿どおり、残業・休出で余力を増やしてから試行するか、タスク×日付 で（段階3前）の数量・日付を人の判断で直してから 段階3 を実行してください。dispatch_trial_shortages.json の op_shortage / as_shortage を確認します。

納期管理ビューが段階2 直後に古い

段階2・段階3 完了直後は自動反映のため一時的に「反映中」になります。失敗時はログの [delivery-calendar] 行を確認し、手動 再読み込み を試してください。

環境変数が反映されない

  • Excel 設定_環境変数 シートの A 列が空・# 始まりでないか
  • Java 側 環境変数 タブで上書きしていないか
  • ポータブル初回起動で既定リセットが走っていないか

マニュアル画像が表示されない

manual/src/images/{tab_key}.png を用意し、リポジトリ直下でマニュアル公開パイプラインを実行してください（manual/README.md 参照）。PNG が無い場合はプレースホルダのままテキストのみです。

サポートに渡す情報

問題報告時は次を添付すると調査が早くなります。

  • 実行・ログの該当箇所（個人情報をマスク）
  • 使用した version.txt の版
  • 段階1/2 の直前にログへ出た環境変数一覧
  • 再現手順（入力ブックは匿名化したコピーでも可）
