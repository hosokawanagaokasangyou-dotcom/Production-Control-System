1. はじめに

本システムについて

工程管理 AI デスクトップは、加工計画データとマスタ情報をもとに 配台計画（設備・担当への割付）を生成・確認・手修正するためのデスクトップアプリです。

  • 段階1 … 加工計画 DATA の読込・成形、配台除外ルールの生成など
  • 段階2 … 配台計画の実行、結果 JSON・設備ガント・配台表の出力
  • 手動修正 … 配台結果を画面上で編集し、再実行に反映

バックエンドの計算は Python（planning_core）が担い、Excel マクロブック（タスク入力ブック）との連携も想定しています。

動作環境

項目 内容
OS Windows 10 / 11（本番想定）。開発時は WSL から起動することもある
実行形態 ポータブル配布（PMD.exe + pm-ai-data）または開発ビルド
Excel マクロブック・xlwings 連携時は Microsoft Excel が必要
ネットワーク タスク入力・実績明細を共有フォルダから読む構成では UNC パス到達性が必要

画面の全体像

メインウィンドウ上部に テーマ・実行進捗・メモリ表示があり、その下が タブ で区切られた各機能です。

既定では次のような並びです（「タブ整理」で並べ替え・グループ化可能）。

区分 主なタブ
日常運用 実行・ログ、配台計画_タスク入力、配台計画手動修正、設備ガント、納期管理ビュー、オペレーターカード
バッジ設定 UIバッジ、プッシュボタン、設備ガント・担当バッジ
環境設定 環境変数、メモリ設定、グローバル設定、ユーザープロファイル
結果情報 計画結果ビューア、段階1 成形結果、材料・製品種類情報、結果_配台表
その他 マスタ読込サマリ、配台不要ルール、特別ルール、実績DATA取得状態 など

用語

用語 意味
タスク入力ブック 配台計画シートなどを含む Excel マクロブック（TASK_INPUT_WORKBOOK）
段階1 / 段階2 計画パイプラインの前処理フェーズと配台実行フェーズ
結果 JSON 段階2 出力の production_plan 等（計画結果ビューア・ガントで参照）
ワークスペース履歴 配台結果のスナップショット保存・復元

初回セットアップの要点

  1. 配布フォルダを所定の場所に展開し、PMD.exe を起動する。
  2. 環境変数 タブで、タスク入力元・マスタ・ネットワークソースのパスを確認する。
  3. 配台計画_タスク入力 で入力ブックとシートを指定する。
  4. 実行・ログ から段階1 → 段階2 の順で実行する。

詳細は 起動と基本操作、日常運用ガイド、配台の流れ を参照してください。
