# 3. 日常運用ガイド（配台システム使い方）

本章は現場オペレーション向けの **一連の手順** を、Word 原稿「配台システム使い方」（2026-05-16 版）と **実装・配台ルール** に照合して整理したものです。画面名・ボタン名は JavaFX アプリの表示に合わせています。

## 全体の流れ

```text
[アラジン] 仮配台・計画/実績の出力
    ↓
[工程管理 AI] 段階1 → タスク入力の確認・編集 → 段階2
    ↓
[確認] 納期管理ビュー / 設備ガント / 配台計画手動修正
    ↓
[任意] 段階3（配台試行）→ Excel・サマリ確認 → [アラジン] 本番反映
```

<div class="callout callout-info">
<p><strong>マスタの前提</strong> … <code>master.xlsm</code>（国分工場では <code>国分masters.xlsm</code> 等）の設定は別資料で完了しているものとします。未設定のときは <strong>マスタ読込サマリ</strong> タブと段階1 ログで不足を確認してください。</p>

</motion>

## 事前準備（アラジン）

### 生産計画入力での「仮」配台

アラジンの **生産計画入力** で、配台日・数量を **仮** として事前入力しておきます。

- この時点の仮数量は **自動配台では無視** されます（段階2 の割付根拠にはなりません）。
- 仮を入れておかないと、後段で **加工計画 DATA から行を吸い出せない** ため、運用上は仮入力が必要です。
- システム配台を本番とする場合、確定後はアラジン側の仮計画も **実績に合わせて更新** する必要があります（後述「納期管理ビュー」参照）。

### 加工計画 DATA の出力（工程別生産計画問い合わせ）

1. アラジンで **工程別生産計画問い合わせ** を開く。
2. **倉庫** … 湖南工場 `520201` / 国分工場 `511101`。
3. その他の条件は社内手順書の画面キャプチャに従い、**問合せ** を実行する。
4. 抽出結果の **列設定は必ず初期化** してから出力する（列ずれ防止）。
5. **フォルダ** で出力先を指定し、**出力** を実行する。

| 工場 | 出力フォルダ（例） |
|------|-------------------|
| 湖南工場 | `\\192.168.0.101\共有フォルダ\湖南工場\湖南共有\生産管理システム\管理システム\●DATA\生産計画問合せ` |
| 国分工場 | `\\192.168.0.101\共有フォルダ\国分工場\国分共有\●配台AIシステム\DATA\計画` |

環境変数 `PM_AI_TASK_INPUT_SOURCE_DIR` でネットワーク代替を使う場合は、上記と同内容が `.pm-ai-cache/network-source/` にコピーされているか **実績DATA・明細の取得状態** で確認します。

### 加工実績明細 DATA の出力（加工実績明細問合せ）

1. **加工実績明細問合せ** を開く。
2. **倉庫** … 計画と同じコード（湖南 `520201` / 国分 `511101`）。
3. **加工日付** … できるだけ短い期間（**1 日分** の実績があれば足りることが多い）。
4. 列設定を **初期化** し、**問合せ** → **フォルダ** 指定 → **出力**。

| 工場 | 出力フォルダ（例） |
|------|-------------------|
| 湖南工場 | `\\192.168.0.101\共有フォルダ\湖南工場\湖南共有\002  加工G\●検査表作成\加工実績明細DATA\` |
| 国分工場 | `\\192.168.0.101\共有フォルダ\国分工場\国分共有\●配台AIシステム\DATA\実績` |

## 工程管理 AI デスクトップ：段階1

<!-- MANUAL_SNAP:run -->

<ol class="flow-steps">
<li><strong>起動</strong> … 配布の <code>PMD.exe</code>（または開発ビルド）を起動する。</li>
<li><strong>実行・ログ</strong> タブを開き、必要なら <strong>ログをクリア</strong> で古いログを消す。</li>
<li><strong>段階1 実行</strong> を押す。ネットワークソースをキャッシュで代替しているときは、段階1 ボタン左にバッジが出ることがあります。</li>
<li>正常終了するとダイアログが出て、自動的に <strong>配台計画_タスク入力</strong> タブへ切り替わる。</li>
</ol>

### 段階1 で行われること（ロジック要約）

| 処理 | 内容 |
|------|------|
| 読込 | 加工計画 DATA・マスタ・配台計画シートの統合 |
| 配台試行順番 | 段階1 出力時に **1…n を自動付与**（手入力の古い順序は引き継がない） |
| 配台不要ルール | `master.xlsm` の設定等から JSON 化（`配台不要ルール` タブで確認可） |
| 全数未加工 | 換算数量 &gt; 0 かつ実加工数・未加工がともに 0 の行は **全数未加工** と解釈 |

詳細は [配台の流れ（段階1・段階2・段階3）](workflow-dispatch.md) とリポジトリ内 `code/要件定義/配台ルール.md` を参照してください。

## 配台計画_タスク入力：確認と編集

<!-- MANUAL_SNAP:planInput -->

### 画面の見方

- 上部 **操作・ソース** … ブックパス・シート名・**読込** / **保存**・**段階2 実行**。
- 下部 **データ表** … 配台対象行のスプレッドシート。列ヘッダの **▽** で Excel 風フィルタが使えます。

**ヒント** … 「操作・ソース」のアコーディオンを一度閉じると、表の表示領域が広くなります。

### フィルタの例（加工内容）

**加工内容** 列のフィルタ（▽）で、例えば `EC` のみ、`EC,検査` のみ、などに絞り込めます。フィルタ適用中は **該当行だけ** が表示されます（Excel と同様の操作感）。

業務ルール上、同一依頼で `EC` → `検査` の順序がある場合、**EC の残ロールがある間は検査に自動割付されない** 制約があります（`配台ルール.md` §A-1 / §B-2）。フィルタで工程ごとに件数を確認すると安全です。

### 配台試行順番の変更

自動配台は **配台試行順番** の昇順で設備・担当を割り付けます（環境変数 `STAGE2_DISPATCH_FLOW_TRIAL_ORDER_FIRST` が有効なときの既定）。

- **依頼NO** 付近の行をドラッグ＆ドロップで入れ替え、優先したい依頼を上にします。
- ↑ / ↓ ボタンでも行を移動できます。
- **すべての行** に正の整数が入っているときだけ、シート上の順序がそのまま段階2 に反映されます。1 行でも欠損があると **自動再計算** にフォールバックします。

### 配台不要（行の除外）

**配台不要** 列のセルを **ダブルクリック** すると、配台対象外として `yes` 相当の値が入ります（セルは **赤背景・白字** で強調）。

有効な値の例: `yes` / `1` / `はい` / `○` など（`配台計画除外` を含む文字列も除外扱い）。段階2 のタスクキューに **載りません**。

### 上書き列（速度・原反投入日）

| 列 | 操作 | 備考 |
|----|------|------|
| 加工速度_上書き | ダブルクリックで編集 | 自動配台の速度計算に反映 |
| 原反投入日_上書き | ダブルクリックで編集 | **アラジン上の元データは書き換えません**（本システム内の計画用） |

### 保存と段階2

1. **操作・ソース** を開き、表の編集を **保存** する。
2. **当日は配台しない（翌日以降から計画）** にチェックが入っていることを確認する（当日分を計画に含めない運用）。
3. **段階2 実行** を押す。

<div class="callout callout-warn">
<p><strong>未保存のまま段階2 は実行できません。</strong> 保存後、自動的に <strong>実行・ログ</strong> タブへ切り替わり段階2 が走ります。実行中は他タブが灰色になることがあります（ツールバーの <strong>中断</strong> で停止可）。</p>

</div>

### 段階2 実行中・完了後

- **Gemini API** を使う設定では、呼び出しに **数分以上** かかることがあります。ログ末尾を確認してください。
- 正常終了すると **段階2 完了** ダイアログが出ます。
- **実行・ログ** に `production_plan` / `member_schedule` の最新パスが表示され、**エクセルを開く** で Excel 成果物を開けます。

## 結果の確認とアラジン連携

### 納期管理ビュー

<!-- MANUAL_SNAP:deliveryCalendarView -->

- **納期判定** 列で **NG** の行を確認する。
- 対応の要点 … **回答納期** を、**加工終了日時** の **次稼働日** にアラジン上で設定する（現場手順）。
- 本ビューでは **アラジン計画** と **システム配台（シス配台）** を並べて比較できます。
- 段階2 または段階3 完了直後は、古い表を見せないよう **自動再読込** される設計です（手動「再読み込み」前の誤判断防止）。

**納期基準の優先**（ロジック） … 回答納期が入っていればそれを基準に、空なら **指定納期** を使います（`配台ルール.md` §2.3 改定参照）。

### 設備ガント（グラフィック）

<!-- MANUAL_SNAP:equipmentGanttGraphic -->

設備別のタイムラインで、割付の重なり・空白・担当表示を **グラフィカルに評価** します。印刷・倍率は環境変数 `GANTT_*` 系とガントタブの UI 設定に依存します。

### 配台計画手動修正（段階2 結果の精査・段階3）

<!-- MANUAL_SNAP:dispatchInteractive -->

| 子タブ／表 | 用途 |
|------------|------|
| **工程・機械×日** | 機械名ごとの日別合計加工量。設備負荷の妥当性確認 |
| **タスク×日付** | 依頼NOごとに「いつ・何 m」配台されているか。ダブルクリック編集・ドラッグで日付間移動 |

**段階3** ボタン（画面上は「段階3」）が **インタラクティブ配台試行** です。段階2 とは別エンジン経路で、表の **当日配台数量（段階3前）** を満たすようタイムラインを組み立てます。

<div class="callout callout-warn">
<p><strong>段階3 の制約（運用上重要）</strong></p>
<ul>
<li>試行は <strong>最初に配台数が現れる日</strong> から始まりますが、<strong>勤怠（master の人員カレンダー）</strong> に縛られる点は段階2 と同様です。そのため <strong>当日に指定した加工数を必ず満たせるとは限りません</strong>。</li>
<li>不足時は <strong>残業・休出で余力を増やしてから</strong> 段階3 を実行する、または <strong>タスク×日付</strong> で人の判断を先に反映してから試行する、という運用が docx 原稿でも推奨されています。</li>
<li>試行後のセルは <strong>（段階3前）</strong> と <strong>（段階3後）</strong> の2行表示になり、<strong>（段階3後）</strong> は編集しません。</li>
<li>人員不足・割当不可は <code>dispatch_trial_shortages.json</code> 等でモーダル表示されます。</li>
</ul>
<p>将来版では、数量を死守したうえで「いつ・どれだけ残業すれば足りるか」をシステムが示す拡張が docx に記載されています（現行未実装の場合あり）。</p>
</motion>

### 段階3 実行後

1. **保存 (JSON+xlsx)** で手修正を確定する（試行ボタンは未保存でも試行開始時に同期するが、運用では保存を推奨）。
2. **実行・ログ** の **エクセルを開く**（`production_plan` / `member_schedule` / サマリ_AI配台）で Excel 側を確認する。
3. **納期管理ビュー** でも同系の数量・納期を確認できる。
4. **アラジンへ転記** … 32 インチ程度のワイドモニターで、アラジンと本アプリを **上下または左右に並べ**、シス配台結果を生産計画入力へ写す。

## チェックリスト（1 日の締め）

| # | 確認項目 |
|---|----------|
| 1 | 段階1・段階2 がログ上 exit 0 で終了している |
| 2 | タスク入力表を **保存** 済み |
| 3 | 納期管理ビューで NG 行に手順どおり対応した |
| 4 | ガント・手動修正表で設備・人の偏りがない |
| 5 | 段階3 を使った場合、未達・不足ダイアログを解消または記録した |
| 6 | アラジン本番計画をシス配台と整合した |
