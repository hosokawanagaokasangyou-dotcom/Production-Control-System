# 7. Excel 連携とサマリ

## Excel の役割

本システムは **Excel マクロブック** をハブに据えた運用を想定しています。

| ブック・シート | 役割 |
|----------------|------|
| タスク入力ブック（`TASK_INPUT_WORKBOOK`） | 配台計画シート、加工計画 DATA 参照、VBA/xlwings から Python 起動 |
| マスタブック（`master.xlsm` 等） | 設備・速度・組み合わせ表など |
| 設定_環境変数 シート | 変数名・値を Python 起動前に環境へ反映 |
| 結果_配台表.xlsx | 段階2 後の配台表（JSON から export） |

Java デスクトップから段階1/2 を実行する場合も、最終的な計画シート・マクロは Excel 側と整合している必要があります。

## サマリ_AI配台（Excel）

`code/サマリ_AI配台.xlsm`（または `.xlsx`）は、配台・AI 利用状況などを **集計・可視化** するサマリ用ブックです。主な用途の例:

- 段階2 やパターン別バッチの **試行結果・スコア** の一覧
- **Gemini トークン・料金** など AI 利用サマリの反映（メインシート P 列等）
- 納期管理ビュー向け **(シス配台)** 表示用数量の参照

> セル配置・マクロ名はブックのバージョンに依存します。ブック内の説明シートまたは VBA コメントを正としてください。

### パターン別段階2 サマリ

環境変数 `DISPATCH_PATTERN_STAGE2_SUMMARY_SHEET` でサマリシート名を指定できます。バッチ実行後:

1. サマリの **採用パターン ID**（例: B3）を入力
2. バッチ出力ルート（例: B2）が有効であることを確認
3. マクロまたは Python コマンドで **配台計画シートへ試行順を反映**

## xlwings と自動計算

`XLWINGS_SUSPEND_AUTO_CALCULATION` が有効（既定）のとき、一括書き込み前後で Excel の計算モードを制御し、性能と安定性を確保します。書き込み後に計算が戻らない場合は Excel を一度保存・再計算してください。

## VBA 変更時の確認

マクロを編集した場合は、Excel VBE で **デバッグ → VBAProject のコンパイル** を必ず実行し、エラーが無いことを確認してから本番データで段階2 を回してください。

## Java デスクトップとの使い分け

| 操作 | 推奨場所 |
|------|----------|
| 日常の段階1/2・ログ確認 | Java デスクトップ |
| セル単位の帳票調整・社内定型マクロ | Excel |
| 配台結果の GUI 手修正 | デスクトップ「配台計画手動修正」 |
| 複数パターン一括試行 | Excel マクロ / バッチ + サマリシート |
