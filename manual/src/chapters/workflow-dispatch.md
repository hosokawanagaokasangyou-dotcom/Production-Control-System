# 4. 配台の流れ（段階1・段階2・段階3）

## 概要

配台処理は **段階1（前処理）** → **段階2（自動配台）** → （任意）**段階3（インタラクティブ配台試行）** の三層です。段階1・2 は **実行・ログ** または **配台計画_タスク入力** から起動し、バックグラウンドで Python（`planning_core`）が動きます。

```text
[タスク入力・加工計画DATA] → 段階1 → [成形・除外ルール等]
    → タスク入力で編集・保存 → 段階2 → [配台結果 JSON / xlsx / ガント]
    → （任意）手動修正タブで段階3 → [タイムライン調整・不足一覧]
```

業務仕様の正本は `code/要件定義/配台ルール.md` です。日常のクリック手順は [日常運用ガイド](daily-operations.md) を参照してください。

## 段階1（前処理）

### 目的

- 加工計画 DATA の読込・検証・成形
- 配台不要ルール（JSON）の生成・更新
- **配台試行順番** の自動付与（1…n）と段階2 向け中間成果の整備

### 操作

1. **配台計画_タスク入力** で入力ブックとデータソースを正しく指定する（初回のみ。以降はセッション復元可）。
2. **実行・ログ** で **段階1 実行** する。
3. 完了ダイアログ後、自動で **配台計画_タスク入力** タブへ遷移する。
4. **段階1 成形結果** タブで中間出力を確認する（必要に応じて）。

### ロジック上の要点

| 項目 | 挙動 |
|------|------|
| 配台試行順番 | 段階1 抽出直後に列をクリアし、マスタ need・特別ルール等を踏まえて **自動採番**。既存シートの手入力順は引き継がない |
| 全数未加工 | 換算数量 &gt; 0、実加工数 = 0、未加工 = 0 → **全数未加工**（未加工量 = 換算数量） |
| 加工最小長さ | 配台に使う加工の最小長さ ≥ **ロール単位の長さ**（下回る設定は不可） |
| 配台不要 | `master.xlsm`「設定_配台不要工程」または JSON ルールで段階2 キューから除外 |

### よくある入力ミス

- 加工計画 DATA のパスがネットワーク未接続のまま
- マスタブック（`MASTER_WORKBOOK_FILE`）が見つからない
- 列設定未初期化のアラジン出力をそのまま読んだ（列ずれ）

## 段階2（配台実行）

### 目的

- 配台計画シートに基づく設備・担当割付（日次マルチパス）
- `production_plan` 等の結果 JSON 出力
- 結果_配台表 xlsx、設備ガント用データの生成

### 操作

1. 段階1 が成功していることを確認する。
2. **配台計画_タスク入力** で表を確認・編集し **保存** する。
3. **当日は配台しない** のチェックを運用どおりに設定する。
4. 同タブの **段階2 実行** を押す（実行中は **実行・ログ** に切り替わる）。
5. 完了後、出力フォルダの JSON パスがログ・実行タブに反映される。

### 手動修正・未保存との関係

- **配台計画手動修正** タブに未保存があると、タスク入力側の段階2 がブロックされる場合があります。
- 手動修正 JSON を編集したあとは **保存** または **再読み** してから段階2 を再実行してください。

### ロジック上の要点（段階2）

| テーマ | 要約 |
|--------|------|
| 試行順 | 既定は **配台試行順の昇順** で日内マルチパス（`STAGE2_DISPATCH_FLOW_TRIAL_ORDER_FIRST=1`） |
| §B-2 EC→検査 | 同一依頼で EC を優先し、検査はロール枠・リワインドで制御 |
| 納期基準 | **回答納期** → なければ **指定納期** |
| 終業直前 | 残り時間・残ロールが閾値以下だと **新規開始を繰り下げ／却下**（`ASSIGN_END_OF_DAY_DEFER_*`） |
| 休憩 | 休憩帯内開始や休憩までの実働不足は **休憩終了まで繰り下げ** |
| 特別ルール | L 番号ルールは `特別ルール列挙.md` とアプリ「特別ルール」タブと対応 |

## 段階3（インタラクティブ配台試行）

段階3 は **配台計画手動修正** タブの **段階3** ボタンから実行します（`dispatch_interactive_trial.py`）。段階2 の結果 JSON を入力に、**タスク×日付** 表の **（段階3前）** 数量を満たすタイムラインを組み立てます。

### 操作

1. 段階2 が成功し、手動修正タブで JSON を **再読み** できる状態にする。
2. 必要なら **タスク×日付** で数量をドラッグ／ダブルクリック編集する（**（段階3前）** のみ）。
3. **段階3** を実行し、ログウィンドウで進捗を確認する。
4. 未達・人員不足があればモーダルと **配台数量未達** パネルを確認する。
5. **保存 (JSON+xlsx)** → 納期管理ビュー・設備ガント・サマリ Excel を更新確認する。

### ロジック上の要点（段階3）

| 項目 | 挙動 |
|------|------|
| 数量検証 | 依頼NO×機械名の合計が入力 JSON と一致することを検証 |
| 機械カレンダー | `*` / `＊` / `※` セルは占有。スロット行のない時刻は配台不可 |
| 人員不足 | `op_shortage`（人数不足）と `as_shortage`（組合せ不可）を区別して記録 |
| 二相（任意） | `PM_AI_STAGE3_TWO_PHASE=1` 等で設備フェイズ→人割当フェイズに分割可能 |
| 勤怠 | master の勤怠・休憩に拘束されるため、**当日指定数量を必ず満たせるとは限らない** |

## 結果の確認

| タブ | 確認内容 |
|------|----------|
| 計画結果ビューア (JSON) | 段階2 成果 JSON の構造・シート相当ビュー |
| 結果_配台表 (JSON) | 配台表形式の結果 |
| 設備ガント（グラフィック） | 時間軸・設備行の可視化 |
| 納期管理ビュー | 計画と実績・納期の比較（段階3 後はシス配台のみ再反映も可） |

## パターン別段階2（上級）

複数の **配台試行順パターン** を一括試し、スコア付きサマリを Excel に出す運用があります（環境変数 `DISPATCH_PATTERN_*` 系）。通常の単発段階2 とは別バッチです。サマリシートで採用パターンを選び、マクロまたはコマンドで本番シートへ反映します。

詳細は [Excel 連携とサマリ](excel-integration.md) および `code/設定_環境変数_雛形.tsv` の該当キーを参照してください。

## ワークスペース履歴

**配台ワークスペース履歴** タブでは、過去の配台結果スナップショットを保存・復元できます。列順やガント表示状態の復元に使います。
