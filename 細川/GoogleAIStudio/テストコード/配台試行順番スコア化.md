## 処理順（変更後）

1. **パターン用の並べ**（納期順・機械名＋納期・またはシャッフル）  
2. **`_reorder_task_queue_b2_ec_inspection_consecutive`**（§B-2/3 EC 隣接）  
3. **`_reorder_task_queue_slit_sec_consecutive`**（スリット→SEC）  
4. **`_reorder_task_queue_in_progress_front_stable`**（**加工途中 `in_progress` を最後に前へ**。`list.sort` は安定なので、加工途中同士・未着手同士の相対順はその直前の並びを維持）  
5. **`_assign_sequential_dispatch_trial_order`**（1..n）

## コード上の追加

- **`_reorder_task_queue_in_progress_front_stable`** … キー `(0 if in_progress else 1)` のみで安定ソート  
- **`_finalize_dispatch_trial_pattern_queue_after_pattern_sort`** … 上記 2〜5 をまとめた共通後処理  
- **`_apply_dispatch_trial_pattern_sort_pipeline`** … パターンソートのあと上記ヘルパのみ呼ぶよう変更  
- **ランダムパターン** … シャッフル後も同じ `_finalize_dispatch_trial_pattern_queue_after_pattern_sort` を使用  
- シート1行目の説明文を、上記の「加工途中は最後」の内容に合わせて更新  

**補足:** 段階2本体の `_apply_dispatch_trial_order_for_generate_plan` は、従来どおり **`_generate_plan_task_queue_sort_key` の先頭で加工途中を優先**したうえで EC・スリットの順です。パターン一覧だけ挙動を変えています。本番の段階2も「EC・スリットのあとに加工途中」を揃えたい場合は、その旨を指定してください。
