あなたは工場の配台システム用です。次の「配台不能の説明」を、タスク1行を判定する機械可読ルールに変換してください。

【出力】先頭が { で終わりが } の JSON オブジェクト1つのみ（説明・マークダウン禁止）。

【スキーマ version は必ず 1】
1) 常に配台不要（説明が条件なしで外す意味）のとき:
{"version":1,"mode":"always_exclude"}

2) 列の条件で配台不要とするとき:
{"version":1,"mode":"conditions","require_all": true または false,"conditions":[ ... ]}

conditions の各要素:
- {"column":"列名","op":"empty"} … セルが空
- {"column":"列名","op":"not_empty"}
- {"column":"列名","op":"eq","value":"文字列"} / ne / contains / not_contains / regex（正規表現）
- {"column":"列名","op":"gt"|"gte"|"lt"|"lte","value":数値} … 数値比較（列は数として解釈）

【使用可能な列名のみ】（これ以外は使わない）:
タスク加工効率, データ抽出日, 依頼NO, 優先度, 加工完了区分, 加工工程の決定プロセスの因子, 加工速度, 加工速度_上書き, 加工開始日_指定, 加工開始時刻_指定, 原反投入日, 受注数, 回答納期, 実出来高, 実加工数, 工程名, 必要人数, 担当OP_指定, 指定納期, 指定納期_上書き, 換算数量, 機械名, 特別指定_備考, 製品名

【説明文】
※配台計画側で自動処理：同一依頼NOかつ同一機械名が複数行あるときのみ、工程「分割」行の配台不要を空欄ならyesにする。手入力の配台不要は上書きしない。
